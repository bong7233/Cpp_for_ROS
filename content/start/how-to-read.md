# 0.1 이 책을 읽는 법

::: lead
이 책의 목표는 하나다. **ROS 2를 C++만으로 자유롭게 구현하는 로봇 애플리케이션 개발자**가 되는 것. 이 절은 그 목표까지 가는 지도이자 사용 설명서다. 10분이면 읽는다. 건너뛰지 마라 — 특히 이 책의 두 기둥인 **위젯 조작법**과 **타이핑 원칙**이 여기 있다. 이 책을 어떻게 쓰느냐가 도달 지점을 결정한다.
:::

## 이 책이 서 있는 자리

시중의 C++ 자료는 대개 세 종류다.

| 종류 | 다루는 것 | 빠진 것 |
| --- | --- | --- |
| 입문서 | 문법, 클래스, 간단한 예제 | 메모리에서 실제로 무슨 일이 일어나는지 |
| 심화서 (Effective 시리즈 등) | 관용구, 함정, 모범 사례 | 처음 배우는 사람을 위한 발판, 응용 분야 |
| 분야별 전문서 (ROS 2 등) | 해당 프레임워크 API | 언어의 기반, 왜 그 API가 그렇게 생겼는지 |

셋을 다 사서 읽으면 될 것 같지만, 실제로는 잘 안 된다. **연결이 끊기기 때문**이다. 입문서의 포인터가 왜 `rclcpp`의 `SharedPtr` 콜백 시그니처로 이어지는지, 심화서의 이동 시맨틱이 왜 메시지 파이프라인의 제로 카피와 닿아 있는지, executor가 왜 스레드와 락의 문제인지 — 이런 질문의 답은 세 책 **사이**에 있다.

이 책은 그 사이를 메우려고 쓴다. 그래서 순서가 이렇다.

::: note 이 책의 논리 구조
**언어의 골격**(Part I)을 세우면 → **메모리와 소유권**(Part II)이 C++의 심장임을 알게 되고 → **다형성·템플릿·STL**(Part III~V)이 그 위의 표현 도구로 보이고 → **동시성**(Part VI)으로 로봇 제어 루프의 전제를 갖추고 → **도구와 성능**(Part VII~VIII)으로 프로덕션급 개발 사이클을 만들고 → **로보틱스 수학**(Part IX)을 C++로 구현한 뒤 → **ROS 2**(Part X)와 **로봇 SW 스택**(Part XI)이 이 모두를 로봇 위에서 조립하고 → **실전·포트폴리오**(Part XII)로 마무리한다.
:::

## 지금 당신의 위치

이 책은 **일반 프로그래밍 경험이 있는 사람**을 기준으로 쓴다. 변수, 함수, 조건문, 반복문이 무엇인지 알고, 어떤 언어로든 프로그램을 짜 본 적이 있으면 충분하다. C++ 특유의 것 — 포인터, 수동 메모리 관리, 템플릿, RAII — 는 모른다고 가정하고 바닥부터 다룬다.

그렇다고 Part I을 건너뛰지 마라. Part I은 "변수 선언하는 법"을 가르치지 않는다. **소스 파일이 실행파일이 되기까지 컴파일러와 링커가 실제로 무엇을 하는지**를 다룬다. 아래 코드가 왜 이런 결과를 내는지 3초 안에 설명할 수 있다면 1.1절을 훑고 넘어가도 좋다. 못 하겠다면, 거기가 당신의 출발점이다.

```cpp title="이 코드는 컴파일될까?"
#include <iostream>

int add(int a, int b);          // 선언만 있고 정의가 없다

int main() {
    std::cout << add(1, 2) << "\n";
    return 0;
}
```

::: answer 답이 궁금하다면
**컴파일은 통과하고, 링크에서 실패한다.** `add`의 선언이 있으므로 컴파일러는 호출 코드를 만들 수 있다. 하지만 링커가 `add`의 실제 몸체를 찾지 못해 `undefined reference to 'add(int, int)'` 를 뱉는다. "컴파일 에러"와 "링커 에러"는 완전히 다른 단계에서 나는 완전히 다른 에러다 — 이 구분이 [1.1 컴파일 모델](#/compile-model)의 주제이고, ROS 2에서 라이브러리 링크가 꼬였을 때 당신을 구해줄 지식이다.
:::

## 이 책의 표기 규칙

### 인터랙티브 위젯 — 이 책의 핵심 도구

공간적인 개념(스택/힙, 포인터, 소유권, 스레드 인터리빙)은 글 대신 **조작 가능한 위젯**으로 보여준다. 아래가 위젯이다. 지금 직접 만져 보라.

::: widget demo
{ "values": [3, 1, 4, 1, 5, 9, 2, 6], "title": "위젯 조작 연습" }
:::

모든 위젯의 조작법은 같다.

- **다음 ▶ / ◀ 이전** — 한 스텝씩 진행·되감기. 위젯의 기본 사용법이다.
- **▶ 재생** — 자동으로 스텝을 넘긴다. 한 번 흐름을 본 뒤에는 수동 스텝을 권한다.
- **슬라이더** — 아무 스텝으로나 점프.
- 위젯 아래 **설명 줄**이 지금 스텝에서 일어난 일을 말해 준다. 그림과 설명을 같이 읽어라.

위젯은 눈요기가 아니다. **스텝을 앞뒤로 오가며 "왜 이 상태가 됐는가"를 스스로 설명할 수 있을 때** 그 개념이 당신 것이 된다. 뒤에서 만날 스택/힙 위젯, 포인터 위젯, 스레드 타임라인 위젯이 전부 이 방식이다.

### 코드 블록

컴파일 가능한 완결된 코드는 이렇게 나온다. 오른쪽 위 **복사** 버튼이 있지만, 잠시 뒤 설명할 이유로 **누르지 않기를 권한다.**

```cpp title="greet.cpp"
#include <iostream>
#include <string>

std::string greet(const std::string& name) {
    return "hello, " + name;
}

int main() {
    std::cout << greet("hexpider") << "\n";
    return 0;
}
```

터미널 세션은 이렇게 표시한다. **`$` 로 시작하는 줄이 명령**이고, 나머지는 실제 출력이다.

```console
$ g++ -std=c++20 -Wall -Wextra greet.cpp -o greet
$ ./greet
hello, hexpider
```

주목해야 할 줄은 강조해 둔다. 아래에서는 4번 줄이 핵심이다.

```cpp title="함정: 댕글링 레퍼런스" {4}
const std::string& make_name() {
    std::string local = "temp";
    return local;               // 지역변수의 레퍼런스를 반환한다
}                               // local 은 여기서 파괴된다 — 반환된 레퍼런스는 시체를 가리킨다
```

이 책의 코드는 전부 **C++20, `-Wall -Wextra` 에서 경고 없이 컴파일되는 것**을 기준으로 한다. 조각 코드는 조각임을 밝힌다.

::: perf 성능

### 용어

기술 용어는 **한국어 + 영어 병기**로 처음 나올 때 한 번 표기한다. 예: 미정의 동작(undefined behavior, UB), 자원 획득은 초기화(RAII). 이후에는 문맥에 따라 편한 쪽을 쓴다. 영어 문서·컴파일러 에러 메시지를 읽어야 하는 분야이므로 원어를 지우지 않는다. 왼쪽 아래 **용어 사전**에서 모든 용어의 최초 정의 위치를 찾을 수 있다.

## 어떻게 읽을 것인가

### 원칙 1 — 손으로 치지 않은 코드는 읽지 않은 것이다

이 앱에는 코드 실행 기능이 없다. **일부러 뺐다.** 실행은 당신의 실제 IDE와 컴파일러에서 한다 — 이것이 이 책의 학습 방식 그 자체다.

이유는 단순하다. C++에서 타이핑 실수는 낭비가 아니라 훈련이다. 세미콜론을 빠뜨리고, 헤더를 안 붙이고, `const`를 빼먹을 때마다 컴파일러가 에러 메시지를 뱉는다. **그 메시지를 읽고 고치는 반복이 C++ 실력의 절반이다.** 복사-붙여넣기는 그 훈련 기회를 전부 버린다. [0.3 첫 빌드와 에러 메시지 읽는 법](#/first-build)에서 이 훈련을 본격적으로 시작한다.

특히 **결과를 예측한 뒤에 실행하라.** "이건 링커 에러가 날 거야" 하고 엔터를 친다. 맞으면 지식이 확인되고, 틀리면 그 순간이 가장 크게 배우는 순간이다.

### 원칙 2 — 듀얼 모니터: 한쪽에 이 앱, 한쪽에 IDE

이 앱은 PC 전용이고, **모니터 두 대(또는 화면 분할)를 전제로 설계했다.** 한쪽에 이 앱을, 다른 쪽에 IDE(VSCode + 터미널)를 띄운다. 절을 읽다가 코드가 나오면 그 자리에서 IDE로 쳐 보고, 위젯으로 이해를 확인하고, 다시 본문으로 돌아온다. 창을 절반 폭으로 줄여도 레이아웃이 깨지지 않게 만들어져 있다.

이동 중 복습은 이 앱이 아니라 **EPUB 파일**로 한다 — 같은 본문에서 위젯만 뺀 순수 텍스트 버전을 폰 기본 e리더로 여는 방식이다. [0.4 학습 워크플로](#/workflow)에서 설정법을 다룬다.

### 원칙 3 — 막히면 표시하고 넘어간다

Part II의 이동 시맨틱이나 Part VI의 메모리 오더 같은 주제는 처음에 안 와닿는 게 정상이다. 각 절 맨 아래 **'이 절을 완료로 표시'** 버튼이 있고, 왼쪽 목차에 체크(✓)가 남는다. 완벽히 이해한 절만 체크하고, 나머지는 비워 둔 채 진도를 나가라. 뒤쪽을 읽다 보면 앞쪽이 저절로 풀리는 일이 매우 흔하다 — vtable을 배우면 가상 소멸자가, executor를 배우면 콜백 그룹이 풀린다.

::: tip 검색을 적극적으로 써라
상단 돋보기 또는 <kbd>/</kbd> 키로 **전체 본문 검색**이 열린다. "vtable", "RAII", "QoS" 처럼 개념 이름으로 찾으면 그 개념을 다루는 모든 절이 나온다. 이 책은 앞에서 뒤로 읽는 것만큼이나 **필요할 때 찾아 읽는 레퍼런스**로 쓰라고 만들었다.
:::

### 원칙 4 — 목적에 맞는 경로를 탄다

전체를 순서대로 읽는 게 이상적이지만, 급한 목표가 있다면 아래 경로를 쓴다.

**기술 면접이 코앞이라면**

1. 전 파트의 🎤 **기술면접 포인트** 상자만 훑는다
2. [12.4 기술면접 C++ 개념 질문 총정리](#/interview-cpp)로 마무리
3. 시간이 남으면 Part II 전체 — 면접 C++ 질문의 절반은 메모리와 소유권이다

**헥사포드(Hexpider) 실전이 급하다면**

1. Part II 전체 — `ros2_control`도 결국 소유권과 수명 문제다
2. Part IV 4.1~4.3 (템플릿) — Eigen과 PCL을 읽으려면 필요하다
3. Part VI (동시성) — 제어 루프와 통신 스레드의 분리가 전부 여기 있다
4. Part IX (로보틱스 수학) → Part X (ROS 2)

**언어만 빠르게 한 바퀴 돌고 싶다면**

Part I → II → III 순서로 읽되, 각 절의 `deep` 상자는 건너뛴다. 두 바퀴째에 `deep`만 다시 읽는다.

::: warn 지름길의 대가
경로를 타면 빨리 도달하지만, 반드시 **되돌아와서 메운다**. Part II를 건너뛰고 Part X로 간 사람은 콜백에 잡힌 `SharedPtr`가 왜 메모리를 안 놓는지, 왜 노드가 소멸돼도 타이머가 도는지 앞에서 멈춘다. API 사용법은 검색으로 해결되지만 **왜 그런지**는 검색으로 안 나온다. 그 차이가 이 책의 존재 이유다.
:::

## 실습 환경

Part 0의 나머지 절에서 환경을 제대로 세운다. 지금은 결론만 알아 두자.

- **C++20** 기준으로 쓴다. C++17 이하와 갈리는 지점은 그때마다 표시한다.
- 컴파일러는 **g++ 13 이상** 또는 **clang 16 이상**. 리눅스(우분투) 기준으로 설명하고, 다른 OS는 [0.2 설치](#/toolchain-setup)에서 다룬다.
- 빌드는 **CMake**, 디버깅은 **gdb**, 메모리 검증은 **ASan/UBSan**. 전부 [Part VII](#/cmake-basics)에서 깊게 다루지만 설치는 0.2에서 미리 한다.
- **Part X(ROS 2)** 는 ROS 2 Humble 기준이다. 우분투 + Docker 환경 구성을 0.2에서 안내한다.

## 이 앱 자체에 대해

이 책은 **HTML 파일 하나와 마크다운 묶음**으로 되어 있다. 서버도, 인터넷도, 빌드 도구도 필요 없다. `index.html`을 더블클릭하면 열린다.

- **왼쪽 목차**: 부(Part)를 접었다 펼 수 있고, 위쪽 칸에서 목차를 걸러낼 수 있다. 완료한 절에는 ✓ 가 붙는다.
- **오른쪽 목차**: 지금 보는 절 안에서의 위치. 화면이 좁으면 숨는다.
- **진도**: 왼쪽 위 원형 그래프. 브라우저 `localStorage`에 저장된다 — 이 PC의 이 브라우저가 진도의 기준점이다.
- **용어 사전**: 왼쪽 아래. 모든 용어의 한 줄 정의와 최초 등장 챕터.
- **테마·글자 크기**: 상단 오른쪽 아이콘.
- **단축키**: <kbd>/</kbd> 또는 <kbd>Ctrl</kbd>+<kbd>K</kbd> 검색, <kbd>[</kbd> 이전 절, <kbd>]</kbd> 다음 절, <kbd>Esc</kbd> 닫기.

## 시작하기 전에

마지막으로 하나만.

이 책은 길다. 100개가 넘는 절이 있고, 전부 읽으면 웬만한 전공 서적 몇 권 분량이다. **끝까지 읽는 것이 목표가 아니다.** 목표는 *"모르는 것이 나왔을 때 어디를 봐야 할지 아는 상태"*, 그리고 *"세그폴트가 나도 당황하지 않고 한 층 아래로 내려가서 확인할 수 있다는 자신감"* 이다.

C++은 추상화의 뚜껑을 열 수 있는 언어다. 뚜껑을 열면 스택 프레임이 보이고, vtable이 보이고, 캐시 라인이 보인다. 이 책은 그 "한 층 아래"를 매번 위젯과 함께 같이 내려가 주는 책이다.

그럼 시작하자.
